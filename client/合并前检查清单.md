# Client 合并前检查清单

## 目的

用于在 `codex/phase-one-client-round2` 合回 `main` 前，
给出一份正式、可执行、可复盘的 merge 判断依据。

## 通过标准

以下项目全部满足，才可判定“可合并”：

- 自动化守门通过：
  - backend `diagnose`
  - backend `contract drift`
  - backend `acceptance`
  - client headless project smoke
  - client headless main-scene smoke
  - client headless online smoke
- 真实 backend 在线主链至少有一份在线验证记录：
  - GUI 人工联调记录
  - 或明确标注为“等价在线 smoke 记录”
- 最终合并前仍需补齐一份 GUI 窗口内逐页人工联调记录；未补齐前只能进入最终合并检查前状态，不能直接按“最终可合并”结论收口
- `client/README.md`、`backend/docs/api/README.md`、OpenAPI、根目录正式文档口径未漂移
- 当前新增接口已有 backend 测试覆盖
- 未发现新的阻塞合并问题；若有阻塞，必须先列出并暂停合并

## 固定执行入口

以下命令默认都从仓库根目录执行。

统一命令：

```bash
./client/phase-one-merge-gate.sh
```

该命令固定执行：

1. backend `php ./backend/artisan phase-one:diagnose --profile=interop --json`
2. backend `php ./backend/artisan phase-one:contract-drift-check --json`
3. `godot --headless --path . --quit`
4. `godot --headless --path . --scene res://client/scenes/PhaseOneClient.tscn --quit-after 1`
5. `godot --headless --path . --script ./client/scripts/phase_one_online_smoke.gd -- --base-url=... --bearer-token=...`
6. backend `composer --working-dir=./backend phase-one:acceptance`

默认值：

- `BACKEND_URL=http://127.0.0.1:8000`
- `BEARER_TOKEN=test-token-2001`

说明：

- 第 5 步是“真实 backend 在线 + client API 层”的等价 smoke，不是 GUI 人工点击验证。
- 若本地已有 backend 在线，脚本会直接复用；否则会临时启动 `php ./backend/artisan serve`。

## 真实 backend 在线最小人工联调路径

以下步骤用于窗口内点击验证，不能被 headless smoke 替代：

1. 打开 `res://client/scenes/PhaseOneClient.tscn`
2. 在“环境与 Token”页确认 backend 地址、Bearer Token，并执行 `/readyz`
3. 在角色页读取真实角色列表；若为空，则创建角色
4. 执行“设为当前出战角色”，确认 active 角色切换成功
5. 在主线页按顺序完成：章节 -> 关卡 -> 难度
6. 确认首通奖励状态已刷新，并同步到 prepare / settle
7. 执行 battle prepare，确认自动承接 `battle_context_id` 与 `killed_monsters`
8. 执行 battle settle，确认结果页展示掉落、奖励、入包、首通奖励状态
