# Client 合并前检查清单

## 用途

本文档用于 `codex/phase-one-client-round2` 合回 `main` 前的最后工程质量复核。
它不是开发备忘，而是一份可以直接照着执行、照着勾选、照着追责的 merge gate / smoke / GUI 联调 / 文档契约清单。

当前轮次只聚焦客户端工程质量收口：

- `client/phase-one-merge-gate.sh`
- `client/scripts/phase_one_online_smoke.gd`
- `client/README.md`
- `client/合并前检查清单.md`

本轮不扩新接口，不补未来系统，不改 backend 业务语义。

## 进入最终合并检查的前提

只有以下条件全部满足，才可判定“可以进入合并回 `main` 前最终检查”：

- merge gate 全链路通过
- backend `diagnose / contract drift / acceptance` 全部通过
- client headless project smoke、main-scene smoke、online smoke 全部通过
- `client/README.md`、`backend/docs/api/README.md`、OpenAPI、根目录正式文档无契约漂移
- 两个关键脚本 raw 内容已是正常多行、可 review、可维护文本
- 至少存在一份真实 GUI 窗口联调记录，且不能把 headless smoke 伪装成 GUI 验证
- 没有新的阻塞合并项

## 阻塞合并项定义

出现以下任一情况，直接判定“暂不可合并”：

- `./client/phase-one-merge-gate.sh` 任一步失败
- backend `phase-one:diagnose --profile=acceptance` 失败
- backend `phase-one:contract-drift-check --json` 失败
- backend `composer --working-dir=./backend phase-one:acceptance` 失败
- client headless online smoke 失败，或依赖未就绪
- 真实接口、OpenAPI、README、根目录正式文档之间出现漂移
- 两个脚本 raw 内容重新退回单行 / 压缩状态
- 客户端仍依赖手输默认值替代真实角色列表、真实关卡列表、真实奖励状态
- 本轮没有实际 `git commit` / `git push` 却写成“已完成”

## GUI 人工联调检查项

以下步骤必须在非 headless Godot GUI 进程内完成，不能用 API 层等价 smoke 替代：

1. 打开 `res://client/scenes/PhaseOneClient.tscn`
