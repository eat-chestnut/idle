# Client 合并前检查清单

## 目的

用于在 `codex/phase-one-client-round2` 合回主线前，快速确认当前 phase-one 客户端已经满足“少改、稳改、可联调、可合并”的最低要求。

## 代码与文档

- `client/README.md` 是否已经与当前真实页面流、按钮文案、联调方式一致。
- 是否仍以真实 backend 接口为业务真相，没有回退到本地 mock 成功态。
- 是否没有新增未来系统入口或无关 UI 重构。

## 主流程

- 是否能完成：角色列表/创建 -> 激活角色 -> 角色详情 -> 主线 -> prepare -> settle。
- 当前 active 角色是否真正成为 battle 主路径。
- 章节 -> 关卡 -> 难度是否来自真实 backend 列表，而非手输默认值。
- prepare 成功后是否自动承接到 settle。

## 页面状态

- 角色、背包、穿戴、主线、prepare、settle 是否都有：
  - `loading`
  - `success`
  - `empty`
  - `error`
  - `unauthorized`
- 失败时是否有明确提示，而不是停留在旧成功态。

## 联调能力

- `/readyz?profile=interop` 是否可通过。
- 章节接口探测是否可通过。
- “联调覆盖输入”是否仍可用于异常路径或补测，但默认不抢主流程入口。

## 最小回归

- 已执行 Godot headless smoke check。
- 已人工走通一遍最小主流程。
- 若本轮改动影响 backend 依赖假设，已重新确认 token、OpenAPI、接口示例文档口径。
