

# 《山海巡厄录》接口字段级设计

## 1. 文档目的

本文档用于在《核心开发总纲》《模块拆分与职责清单》《后端服务分层设计》《核心业务链路设计》《第一阶段六条主链清单》《高风险点与约束清单》《接口开发清单》以及各方法级设计文档基础上，进一步下沉到接口字段级，明确《山海巡厄录》第一阶段核心接口的请求字段、响应字段、字段含义、字段来源、字段边界与统一返回规范。

本文档重点不是讨论数据库设计，也不是重复业务链规则，而是解决以下问题：

- 第一阶段每个核心接口到底收什么字段
- 每个核心接口到底回什么字段
- 哪些字段来自正式状态，哪些字段只是摘要
- 字段命名如何统一，避免前后端反复改动
- 哪些字段当前阶段必须返回，哪些字段暂不返回

本文档定位为：

- 第一阶段接口字段级设计文档
- 前后端联调字段基线
- Controller / DTO / Resource 结构设计依据
- 后续接口示例、联调清单与测试用例的重要基线文档

## 2. 接口字段设计总原则

### 2.1 字段必须服务正式主链
接口字段设计必须优先服务六条主链：

1. 配置可读主链
2. 状态可写主链
3. 跌落可结算主链
4. 奖励可防重主链
5. 装备可实例化主链
6. 角色可重算主链

### 2.2 请求字段尽量小，响应字段尽量稳
请求字段只传业务真正必要的信息，不传可以由服务端从正式状态推导出的内容。

响应字段必须尽量稳定，避免因为前端页面临时需要频繁变更核心响应结构。

### 2.3 模板字段与实例字段必须分清
接口字段设计时必须明确：

- 模板字段属于静态配置摘要
- 实例字段属于玩家正式状态摘要

例如：

- `item_id`、`item_name`、`rarity` 属于模板摘要
- `equipment_instance_id`、`bind_type`、`enhance_level` 属于实例状态摘要

### 2.4 摘要字段优先于全量字段
第一阶段接口应优先返回：

- 当前页面真正需要的摘要字段
- 正式状态核对需要的关键字段

不应一开始就把所有数据库字段全部暴露给前端。

### 2.5 字段命名统一使用 snake_case
当前阶段接口字段统一使用 `snake_case`，并与数据库主字段、服务层 DTO 字段尽量保持一致。

### 2.6 统一响应外壳
第一阶段建议所有业务接口统一返回外壳：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

说明：
- `code = 0` 表示成功
- 非 0 表示业务失败或系统失败
- `message` 用于返回摘要信息
- `data` 承载正式业务内容

## 3. 通用字段规范

## 3.1 通用 ID 字段
- `character_id`：角色 ID
- `class_id`：职业 ID
- `item_id`：物品模板 ID
- `equipment_instance_id`：装备实例 ID
- `stage_id`：关卡 ID
- `stage_difficulty_id`：关卡难度 ID
- `monster_id`：怪物模板 ID
- `reward_grant_id`：发奖记录 ID

## 3.2 通用状态字段
- `is_enabled`：启用状态
- `is_active`：当前角色是否启用
- `is_cleared`：本次战斗是否通关
- `grant_status`：奖励发放状态

## 3.3 通用数量字段
- `quantity`：数量
- `before_quantity`：变更前数量
- `after_quantity`：变更后数量
- `add_quantity`：本次增加数量

## 3.4 通用排序字段
- `sort_order`
- `stage_order`
- `difficulty_order`
- `wave_no`

## 3.5 时间字段
第一阶段若接口返回时间字段，统一使用：

- `created_at`
- `updated_at`
- `granted_at`

格式建议统一为后端标准时间字符串，不在本阶段强制增加时间戳双格式。

---

## 4. 通用摘要结构设计

为避免重复定义，第一阶段建议统一使用以下摘要结构概念。

## 4.1 物品摘要 `item_summary`

建议字段：

- `item_id`
- `item_name`
- `item_type`
- `rarity`
- `icon`

## 4.2 装备模板摘要 `equipment_template_summary`

建议字段：

- `item_id`
- `item_name`
- `equipment_slot`
- `rarity`
- `icon`
- `level_required`
- `weapon_category`
- `sub_weapon_category`

## 4.3 装备实例摘要 `equipment_instance_summary`

建议字段：

- `equipment_instance_id`
- `item_id`
- `item_name`
- `equipment_slot`
- `rarity`
- `icon`
- `bind_type`
- `enhance_level`
- `durability`
- `max_durability`

## 4.4 角色摘要 `character_summary`

建议字段：

- `character_id`
- `character_name`
- `class_id`
- `class_name`
- `level`
- `exp`
- `is_active`

## 4.5 槽位摘要 `slot_summary`

建议字段：

- `slot_key`
- `equipped_instance_id`
- `equipment`

其中 `equipment` 为可空的 `equipment_instance_summary` 摘要。

## 4.6 战斗属性摘要 `character_stats_summary`

第一阶段建议字段：

- `attack`
- `physical_defense`
- `magic_defense`
- `hp`
- `mana`
- `attack_speed`
- `crit_rate`
- `spell_power`

## 5. 角色信息接口字段设计

## 5.1 接口定位
接口名称建议：角色信息接口

建议路由：
- `GET /api/characters/{character_id}`

## 5.2 请求字段
路径参数：
- `character_id`：必填，角色 ID

当前阶段不建议额外传复杂查询参数。

## 5.3 响应 data 结构

```json
{
  "character": {
    "character_id": 1001,
    "character_name": "青山",
    "class_id": "class_jingang",
    "class_name": "金刚",
    "level": 1,
    "exp": 0,
    "is_active": 1,
    "unspent_stat_points": 0,
    "added_strength": 0,
    "added_mana": 0,
    "added_constitution": 0,
    "added_dexterity": 0,
    "long_term_growth_stage": null
  }
}
```

## 5.4 字段说明
- `character_id`：正式角色 ID
- `character_name`：角色名称
- `class_id`：职业 ID
- `class_name`：职业名称摘要
- `level`：当前等级
- `exp`：当前经验
- `is_active`：是否当前启用角色
- `unspent_stat_points`：未分配属性点
- 四维加点字段：当前已分配加点
- `long_term_growth_stage`：长期成长阶段预留字段

## 5.5 当前阶段不返回字段
- 全量职业修正系数
- 全量角色战斗属性
- 全量背包信息
- 全量穿戴信息

这些应由其他接口承接。

## 6. 背包列表接口字段设计

## 6.1 接口定位
接口名称建议：背包列表接口

建议路由：
- `GET /api/inventory`

## 6.2 请求字段
查询参数建议：
- `tab`：可选，`stack` / `equipment` / `all`
- `page`：可选
- `page_size`：可选

第一阶段如果不做分页，也可先固定返回全量摘要。

## 6.3 响应 data 结构

```json
{
  "stack_items": [
    {
      "item_id": "mat_wood_001",
      "item_name": "灵木",
      "item_type": "material",
      "rarity": "common",
      "icon": "items/mat_wood_001.png",
      "quantity": 25
    }
  ],
  "equipment_items": [
    {
      "equipment_instance_id": 5001,
      "item_id": "eq_hammer_001",
      "item_name": "青石重锤",
      "equipment_slot": "main_weapon",
      "rarity": "rare",
      "icon": "equipments/eq_hammer_001.png",
      "bind_type": "unbound",
      "enhance_level": 0,
      "durability": 100,
      "max_durability": 100
    }
  ]
}
```

## 6.4 字段说明
### `stack_items`
- 返回当前玩家可堆叠物摘要
- 每条至少包含物品摘要 + `quantity`

### `equipment_items`
- 返回当前玩家装备实例摘要
- 每条至少包含实例 ID、模板摘要、实例状态摘要

## 6.5 当前阶段不返回字段
- 装备实例全量 `extra_attributes`
- 装备模板全量属性详情
- 掉落来源信息

需要时由详情接口承接。

## 8. 装备穿戴接口字段设计

## 8.1 接口定位
接口名称建议：装备穿戴接口

建议路由：
- `POST /api/characters/{character_id}/equip`

## 8.2 请求字段

```json
{
  "equipment_instance_id": 5001,
  "target_slot_key": "main_weapon"
}
```

字段说明：
- `equipment_instance_id`：必填，目标装备实例 ID
- `target_slot_key`：必填，目标槽位键

## 8.3 响应 data 结构

```json
{
  "character_id": 1001,
  "equipped_instance_id": 5001,
  "unequipped_instance_ids": [4002],
  "changed_slots": [
    "main_weapon",
    "sub_weapon"
  ],
  "slot_snapshot": [
    {
      "slot_key": "main_weapon",
      "equipped_instance_id": 5001,
      "equipment": {
        "equipment_instance_id": 5001,
        "item_id": "eq_hammer_001",
        "item_name": "青石重锤",
        "equipment_slot": "main_weapon",
        "rarity": "rare",
        "icon": "equipments/eq_hammer_001.png",
        "bind_type": "unbound",
        "enhance_level": 0,
        "durability": 100,
        "max_durability": 100
      }
    },
    {
      "slot_key": "sub_weapon",
      "equipped_instance_id": null,
      "equipment": null
    }
  ]
}
```

## 8.4 字段说明
- `equipped_instance_id`：本次成功穿上的实例 ID
- `unequipped_instance_ids`：本次被替换或被联动卸下的实例 ID 列表
- `changed_slots`：本次发生变化的槽位
- `slot_snapshot`：换装后的槽位快照摘要

## 8.5 当前阶段要求
- 结果必须基于正式写入成功后的状态返回
- 双手武器联动清理副武器时，必须体现在 `changed_slots` 与 `slot_snapshot` 中

## 17. 错误响应字段设计

第一阶段建议统一错误外壳：

```json
{
  "code": 10001,
  "message": "角色不存在",
  "data": null
}
```

说明：
- `code`：业务错误码
- `message`：错误摘要
- `data`：失败时统一为 `null`

## 17.1 当前阶段建议区分的错误类型
- 角色不存在
- 职业不存在或未启用
- 装备实例不存在
- 装备不归属当前用户
- 槽位不合法
- 等级不足
- 主副武器不兼容
- 关卡难度不存在或未启用
- 战斗上下文无效
- 掉落解析失败
- 奖励已发放
- 奖励发放失败
- 入包失败

---

## 18. 当前阶段不应过早扩展的字段

以下字段当前第一阶段不建议大规模进入正式接口：

- 全量装备模板所有内部计算字段
- 全量职业全部修正系数
- 全量掉落来源明细审计字段
- 全量发奖快照 JSON
- 全量实例扩展属性
- 强化 / 洗练 / 宝石 / 套装 / 经卷 / world level 相关未来字段

原因：
- 当前阶段页面未必需要
- 暴露过多字段会增加前后端耦合
- 后续这些系统尚未正式落地，字段语义还可能调整

---

## 19. 当前阶段结论

1. 第一阶段接口字段必须优先服务六条主链，而不是为临时页面堆字段
2. 请求字段应尽量小，响应字段应尽量稳
3. 模板字段、实例字段、摘要字段、正式状态字段必须严格区分
4. 战斗准备与战斗结算接口应作为第一阶段最核心的复杂字段接口重点收敛
5. 第一阶段不应过早暴露未来成长系统的大量预留字段
6. 后续接口示例、前后端联调、DTO / Resource 编写，应以本文档定义的字段结构为准

本文档可直接作为项目正式《接口字段级设计》文档使用，并作为后续接口示例文档、联调清单与测试用例设计的统一字段基线。
