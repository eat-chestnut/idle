## 3. 页面职责与入口

### 3.1 启动页 / 环境页
职责：
- 显示 backend 地址 / 环境
- 检查 readiness（可选）
- 进入登录或主流程

入口：
- 应用启动

### 3.2 登录 / Token 配置页
职责：
- 设置 bearer token
- 验证最小联调前提

入口：
- 启动页
- token 失效后回退

### 3.3 角色创建页
职责：
- 创建角色
- 进入角色详情

入口：
- 无角色时自动进入
- 手动创建入口（若有）

### 3.4 角色详情页
职责：
- 展示角色基础信息
- 提供进入背包、穿戴、主线的入口

入口：
- 创建成功后进入
- 主导航进入

### 3.5 背包页
职责：
- 展示 stack items
- 展示 equipment instances
- 提供装备查看与穿戴入口

入口：
- 角色详情页

### 3.6 穿戴页
职责：
- 展示 12 个固定槽位
- 展示当前穿戴结果
- 支持 equip / unequip 交互

入口：
- 角色详情页
- 背包页跳转

### 3.7 章节列表页
职责：
- 展示章节
- 进入关卡 / 难度

入口：
- 角色详情页

### 3.8 关卡难度页
职责：
- 展示关卡难度
- 展示首通奖励状态
- 进入 battle prepare

入口：
- 章节列表页

### 3.9 战斗准备页
职责：
- 展示角色战斗摘要
- 展示怪物列表
- 展示 battle_context_id
- 发起结算前的准备展示

入口：
- 关卡难度页

### 3.10 战斗结算结果页
职责：
- 展示掉落结果
- 展示奖励结果
- 展示首通奖励状态变化
- 返回章节或角色页

入口：
- 战斗准备页之后

---

## 4. 推荐页面跳转主链

启动 -> 登录/Token -> 角色创建 -> 角色详情 -> 章节列表 -> 难度页 -> 战斗准备 -> 战斗结算结果

辅助跳转：
角色详情 -> 背包 -> 穿戴
角色详情 -> 穿戴
战斗结算结果 -> 章节列表
战斗结算结果 -> 角色详情

---

## 5. 当前阶段结论

phase-one 不需要很多页面。
重点是把以上 10 个页面按主链打通。
