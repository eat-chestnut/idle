## 3. 页面职责与入口

### 3.1 启动检查页

职责：

- 显示当前 backend 地址与弱联网配置
- 执行一次启动检查
- 记录版本、资源、存档服务可用性快照
- 进入主链

入口：

- 应用启动

### 3.2 角色页

职责：

- 展示当前主角
- 展示当前成长总览
- 提供进入主线、背包、穿戴的入口

入口：

- 启动检查完成后
- 背包 / 穿戴 / Settle 回流

### 3.3 主线页

职责：

- 展示当前推进章节、关卡、难度
- 明确下一场目标
- 进入出战页

入口：

- 角色页
- Settle 后回主线

### 3.4 出战页

职责：

- 确认当前出战角色
- 确认本次目标难度
- 展示本场出战摘要
- 进入 Battle

入口：

- 主线页

### 3.5 Battle 页

职责：

- 展示这一场的战场信息
- 承接本场 battle context
- 明确攻击、受击、击杀、掉落、结束反馈
- 进入 Settle

入口：

- 出战页

### 3.6 Settle 页

职责：

- 展示本场结果
- 展示掉落、奖励、入包承接
- 提示下一步是整理背包、试穿新装备还是回主线

入口：

- Battle 页之后

### 3.7 背包页

职责：

- 展示本轮新增收益
- 区分材料与装备
- 提供前往穿戴与角色页的入口

入口：

- 角色页
- Settle 页

### 3.8 穿戴页

职责：

- 展示 12 个固定槽位
- 展示当前穿戴结果
- 聚焦候选替换
- 支持 equip / unequip

入口：

- 角色页
- 背包页
- Settle 页

---

## 4. 推荐页面跳转主链

启动检查 -> 角色 -> 主线 -> 出战 -> Battle -> Settle -> 背包 -> 穿戴 -> 角色 -> 回主线

辅助跳转：

角色 -> 背包
角色 -> 穿戴
Settle -> 背包
Settle -> 穿戴
Settle -> 角色
Settle -> 主线

---

## 5. 当前阶段结论

当前页面数量不需要继续扩张。
重点是让以上页面按“单机刷图成长主链”收口，而不是继续按“在线联调页”扩散。
