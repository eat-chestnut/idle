# 《山海巡厄录》单机运行时与弱联网边界

## 1. 文档目的

本文档用于正式收口当前游戏端的运行模式边界。

从本轮开始，项目默认方向不再是“phase-one 在线联调客户端”，而是：

**单机为主，弱联网为辅。**

本文档用于统一说明：
- 启动阶段允许的联网边界
- 游戏运行期的本地真相边界
- 存档上传 / 下载边界
- 未来联网能力边界
- 当前客户端开发禁止事项

---

## 2. 启动联网边界

客户端在一次正常启动流程中，默认只允许进行一次后台探测。

这一次联网仅用于：
1. 检查游戏版本
2. 检查配置 / 数据版本
3. 检查资源版本（如当前仓库已声明资源版本）
4. 检查存档上传 / 下载服务是否可用

启动检查的原则：
- 只在启动阶段执行
- 结果必须写入本地 runtime state
- 后续页面不能把这些检查结果当成反复请求后台的理由
- 启动检查失败时，必须明确提示，但不能把游戏主循环重新设计成实时依赖后台

如果当前 backend 尚未暴露完整版本字段：
- 游戏端仍需先建立统一结构
- 缺失字段可标记为 `unknown` / `not_declared`
- 不得因为字段未齐，就把主循环重新绑定到实时 API

---

## 3. 游戏运行期本地真相边界

进入游戏后，当前版本的正式方向是：

**游戏运行期以本地 runtime state 为真相。**

至少以下内容必须逐步切换到本地真相入口：
- 当前角色
- 当前主线目标
- 当前背包
- 当前穿戴
- 当前战斗上下文
- 当前结算结果
- 当前成长承接信息

当前阶段允许的过渡方式：
- 旧接口仍可暂时保留
- 旧页面仍可暂时复用已有 API 调用
- 但主协调器和关键页面必须开始通过本地 runtime state 读写当前状态

不允许的做法：
- 新增页面继续把实时 API 返回直接当最终页面真相
- 继续扩大“点一个页面就打一条正式接口”的结构
- 把启动检查、版本字段、存档能力检查混入普通业务页反复执行

---

## 4. 存档上传 / 下载边界

当前弱联网正式只保留两类运行期能力：
- 存档上传
- 存档下载

这两类能力的边界如下：
- 上传 / 下载是运行期的弱联网附属能力，不是主循环驱动器
- 本地 runtime state 才是当前游戏中的即时真相
- 上传是把本地真相同步出去，不是实时请求后台拿“最终答案”
- 下载是把远端存档载入本地，不是让普通页面持续依赖后台
- 上传 / 下载的对象必须是正式本地存档，而不是页面状态碎片

当前正式推荐的落点拆分为：
- 本地 runtime state：运行中的即时真相
- 正式本地存档对象：可恢复、可上传、可下载的持久化真相
- 弱联网配置与启动快照：单独放在配置层，不混进正式存档对象

当前本地正式存档生命周期应明确收口为：
- 启动时先做一次启动检查，再判断是否存在有效正式本地存档
- 有效存档用于恢复 runtime state；无效或缺失时进入默认新开局初始化
- 角色、主线、背包、穿戴、最近一次 prepare / settle 摘要与必要 UI 焦点可以进入正式本地存档
- battle 表现层的瞬时过程不进入正式本地存档；跨启动只恢复合理的 battle / settle 承接上下文，不伪造“战斗进行中”

当前阶段不提前做：
- 复杂云存档冲突处理
- 多端同时在线一致性系统
- 长连接同步
- 房间 / 协作 / 多人战斗

---

## 5. 未来联网能力边界

未来允许扩展的联网能力主要包括：
- 排行榜
- 交易所
- 其他明确立项的弱联网系统

这些能力的处理原则：
- 单独设计
- 单独划定接口与运行边界
- 单独增加本地与联网的状态切换规则

当前阶段不允许为了“未来也许要做”而提前：
- 把主循环做成重服务端驱动
- 把页面改成实时订阅后台状态
- 引入复杂在线状态系统
- 引入多人协同前提

---

## 6. 当前客户端开发禁止事项

从当前阶段起，客户端开发默认禁止：
- 继续以 phase-one 在线联调客户端作为正式运行模式
- 继续默认“后端实时返回 = 页面主真相”
- 用 merge gate / online smoke 代表未来正式运行形态
- 为排行榜、交易所、多人逻辑提前设计重联网结构
- 把 battle / growth / inventory / equipment / stage 主循环重新强化为实时 API 客户端

允许保留但必须降级的内容：
- merge gate
- online smoke
- backend 契约校验脚本

它们的定位只能是：
- 开发期契约验证工具
- 后端能力 smoke 工具
- 回归检查工具

它们不是：
- 正式运行模式
- 页面真相来源
- 客户端主循环设计基线

---

## 7. 当前阶段结论

当前正式口径为：
- 启动时联网一次
- 运行期以本地 runtime state 为真相
- 后台当前只承担启动检查与存档上传 / 下载边界
- 排行榜、交易所等未来联网能力后续再单独扩展

游戏端后续开发、文档更新、README、skills 与 Codex 协作说明，均应以本文档为统一边界参考。
