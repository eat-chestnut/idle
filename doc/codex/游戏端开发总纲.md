# 《山海巡厄录》游戏端开发总纲

## 1. 文档目的

本文档用于把当前项目的游戏端方向统一收口为：

- 单机为主，弱联网为辅
- 启动只联网一次
- 运行期主循环以本地 runtime state 为真相
- 当前后台只承担启动检查与存档上传 / 下载支撑

本文档重点不是重新定义玩法，而是明确：

- 游戏端当前应该优先搭什么
- 页面以后应围绕什么状态源组织
- 哪些能力属于当前必须收口
- 哪些联网能力应明确延后

---

## 2. 当前阶段目标

当前游戏端目标不是继续做“在线联调客户端”，而是完成一条以本地状态驱动的最小可玩链：

1. 进入客户端
2. 做一次启动检查
3. 锁定当前角色
4. 锁定当前主线目标
5. 进入副本 Battle
6. 回到 Settle
7. 承接掉落、背包、穿戴与角色成长
8. 回主线继续下一轮副本

这条链的正式顺序是：

`主线 -> 出战 -> 副本 Battle -> Settle -> 背包 -> 穿戴 -> 角色 -> 回主线`

当前 Battle 的正式主形态更新为：

- 固定副本大地图刷图
- 进入地图后加载全图怪物
- 玩家可在副本内自由走位
- Boss 速刷与全清路线并存

不再以旧的单场遭遇式 battle 壳作为当前 Battle 页口径。

---

## 3. 开发边界

### 当前必做

- 启动检查页 / 启动检查层
- 本地 runtime / local game state 骨架
- 角色页
- 主线页
- 出战页
- Battle 页
- Settle 页
- 背包页
- 穿戴页
- 当前成长回流
- 存档上传 / 下载入口预留

### 当前不做

- 排行榜
- 交易所
- 多人协作
- 强联网实时状态同步
- 强化
- 洗练
- 宝石
- 套装
- 经卷
- world level
- 商店
- 活动
- 社交
- 复杂成长树
- 大型战斗表现系统重构

---

## 4. 游戏端开发原则

### 4.1 运行期本地为真相

当前客户端运行期的页面组织必须逐步切到本地 runtime state。

本地真相至少包括：

- 当前角色
- 当前主线目标
- 当前背包
- 当前穿戴
- 当前 battle context
- 当前结算结果
- 当前成长承接

### 4.2 后端不再是运行期页面的默认实时真相

后端当前仍约束：

- 启动检查结果
- 现有存档相关能力
- 仍未迁移完成的旧链路兼容

但页面主循环不能继续默认采用：

- 每进一页都再请求一次后端作为真相
- 用在线 smoke 代表未来正式运行模式
- 把联调便捷性放在运行时结构之上

### 4.3 页面按主链逐步打通

先把单条可玩的竖版主链打稳，不先铺很多外围页面。

当前 Battle 页优先承接的不是“遭遇演示”，而是：

- 固定副本地图刷图主循环
- 本地 runtime 驱动的怪物分布与推进
- Boss 击杀后继续清图或立即结算的分叉
- 全清记录回写本地存档

### 4.4 所有页面必须有状态

至少包括：

- loading
- success
- empty
- error
- unauthorized

战斗相关页额外处理：

- preparing
- settling

### 4.5 启动检查只做一次

启动检查当前至少覆盖：

- 游戏 / 应用版本
- 配置 / 数据版本
- 资源版本（若当前项目已声明）
- 存档上传 / 下载服务可用性

检查结果必须写入本地 runtime state，不允许成为运行期反复请求的理由。

---

## 5. 推荐开发顺序

1. 先纠正文档、README 与 skills
2. 建立本地 runtime / local game state 骨架
3. 加启动检查层
4. 让主协调器优先读本地状态
5. 让关键页面开始走本地状态承接
6. 保留旧 API 作为迁移兼容，不继续扩大其职责

---

## 6. 与 backend 的关系

当前 backend 对游戏端的作用已经收缩为：

- 启动检查兼容来源
- 存档上传 / 下载服务
- 迁移阶段旧接口兼容

如果现有 backend 尚未显式提供完整版本字段或存档服务状态字段：

- 客户端应兼容记录为 `unknown` 或 `not_declared`
- 不应因此倒退回“运行期继续以实时 API 为主循环真相”
- 若未来补充正式接口，再单独收口

---

## 7. 当前阶段结论

当前最合理策略是：

- 文档先纠偏
- 运行时状态先本地化
- 启动联网先收缩为一次检查
- 弱联网只保留存档边界
- 页面逐步从“联调客户端”迁移为“单机刷图成长客户端”
- Battle 主形态逐步从“单场遭遇”迁移为“固定副本地图刷图”
