# 《山海巡厄录》接口示例文档

## 1. 文档目的

本文档用于回写第一阶段当前真实存在的前台业务接口示例。

本文档定位为：

- 根目录人读版正式接口示例文档
- 前后端联调、自测、验收时的请求 / 响应参考
- `backend/docs/api/phase-one-frontend.openapi.json` 的人读补充说明

当前维护分工必须明确：

- `backend/docs/api/phase-one-frontend.openapi.json`
  机器可读正式契约，以真实 `routes / requests / resources / tests` 为准
- `doc/codex/接口示例文档.md`
  人读示例文档，只保留当前真实存在的稳定接口

如果代码、OpenAPI 和本文档出现漂移，必须先修 backend 真实实现与契约产物，再回写本文档。

## 2. 公共口径

### 2.1 统一响应外壳

成功响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

失败响应：

```json
{
  "code": 10101,
  "message": "角色不存在",
  "data": null
}
```

### 2.2 认证方式

当前第一阶段前台业务接口统一使用：

```http
Authorization: Bearer <token>
Accept: application/json
```

当前本地最小联调测试 token：

- `Bearer test-token-2001`

该 token 对应 `DatabaseSeeder` 写入的测试用户 `users.id = 2001`。

### 2.3 当前真实前台接口范围

当前 phase-one 前台正式接口共 14 个：

- `GET /api/characters`
- `POST /api/characters`
- `GET /api/characters/{character_id}`
- `POST /api/characters/{character_id}/activate`
- `GET /api/characters/{character_id}/equipment-slots`
- `POST /api/characters/{character_id}/equip`
- `POST /api/characters/{character_id}/unequip`
- `GET /api/inventory`
- `GET /api/chapters`
- `GET /api/chapters/{chapter_id}/stages`
- `GET /api/stages/{stage_id}/difficulties`
- `GET /api/stage-difficulties/{stage_difficulty_id}/first-clear-reward-status`
- `POST /api/battles/prepare`
- `POST /api/battles/settle`

## 6. 装备穿戴接口

### 6.1 接口说明

- 请求方式：`POST`
- 路由：`/api/characters/{character_id}/equip`

### 6.2 请求示例

以下示例基于角色已经先穿戴 `5001(main_weapon)` 和 `5002(sub_weapon)`，再将双手武器 `5003` 穿到主手。

```http
POST /api/characters/1001/equip
Authorization: Bearer <token>
Content-Type: application/json
Accept: application/json

{
  "equipment_instance_id": 5003,
  "target_slot_key": "main_weapon"
}
```

### 6.3 成功响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "character_id": 1001,
    "equipped_instance_id": 5003,
    "unequipped_instance_ids": [
      5001,
      5002
    ],
    "changed_slots": [
      "main_weapon",
      "sub_weapon"
    ],
    "slot_snapshot": [
      {
        "slot_key": "main_weapon",
        "equipped_instance_id": 5003,
        "equipment": {
          "equipment_instance_id": 5003,
          "item_id": "eq_staff_001",
          "item_name": "灵枝法杖",
          "equipment_slot": "main_weapon",
          "rarity": "rare",
          "icon": "equipments/eq_staff_001.png",
          "bind_type": "unbound",
          "enhance_level": 0,
          "durability": 100,
          "max_durability": 100
        }
      },

        "slot_key": "sub_weapon",
        "equipped_instance_id": null,
        "equipment": null
      }
    ]
  }
}
```

### 6.4 失败响应示例

## 13. 战斗结算接口

### 13.1 接口说明

- 请求方式：`POST`
- 路由：`/api/battles/settle`
- `battle_context_id` 必须来自一次成功的 prepare
- 同一 `battle_context_id` 只允许正式结算一次
- 普通掉落与首通奖励会分链处理，最终只在响应层汇总

### 13.2 请求示例

```http
POST /api/battles/settle
Authorization: Bearer <token>
Content-Type: application/json
Accept: application/json

{
  "character_id": 1001,
  "stage_difficulty_id": "stage_nanshan_001_normal",
  "battle_context_id": "battle_ctx_20260320_120000_ab12cd",
  "is_cleared": 1,
  "killed_monsters": [
    "monster_boss_001"
  ]
}
```
