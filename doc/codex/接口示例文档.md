本文档用于在《项目总览》《接口开发清单》《接口字段级设计》《Laravel 代码目录与命名规范》以及各方法级设计文档基础上，进一步给出《山海巡厄录》第一阶段核心接口的请求示例、成功响应示例、失败响应示例与字段使用说明。

本文档不替代接口字段级设计文档，而是作为：

- 前后端联调示例文档
- Codex 实现接口时的直接参考文档
- 测试、自测、验收时的请求 / 响应样例文档
- 后续 Swagger / OpenAPI 文档整理前的项目内部示例基线

本文档当前优先覆盖第一阶段核心接口，不追求一次性把所有非核心接口全部补齐。

## 2. 统一响应外壳示例

第一阶段所有正式业务接口统一使用如下响应外壳。

### 2.1 成功响应外壳

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

### 2.2 失败响应外壳

```json
{
  "code": 10001,
  "message": "角色不存在",
  "data": null
}
```

### 2.3 通用约束

- `code = 0` 表示成功
- 非 `0` 表示业务失败或系统失败
- `message` 为简要说明
- `data` 成功时为对象或列表，失败时为 `null`

## 3. 角色信息接口示例

## 3.1 接口说明

- 接口名称：角色信息接口
- 请求方式：`GET`
- 建议路由：`/api/characters/{character_id}`

## 3.2 请求示例

```http
GET /api/characters/1001
Authorization: Bearer <token>
Accept: application/json
```
