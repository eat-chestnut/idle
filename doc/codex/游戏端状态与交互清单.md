# 《山海巡厄录》游戏端状态与交互清单

## 1. 文档目的

本文档用于统一当前单机优先阶段的页面状态、交互边界和最小反馈规范。

---

## 2. 所有页面必须具备的基础状态

每个页面至少必须处理：

1. loading
2. success
3. empty
4. error
5. unauthorized

不允许：

- 只有成功态
- 出错直接白屏
- 因旧接口失败就让整条本地主循环失去承接

说明：

- `unauthorized` 当前主要仍用于开发环境、弱联网入口和旧接口兼容
- 运行期主页面组织应优先依赖本地 runtime state，而不是把所有状态都退回成“重新请求失败”

---

## 3. 页面级状态要求

### 3.1 启动检查页

状态：

- loading
- success
- error
- unauthorized

交互：

- 执行一次启动检查
- 显示版本与服务快照
- 检查完成后进入主链

### 3.2 角色页

状态：

- loading
- success
- empty
- error
- unauthorized

交互：

- 展示当前主角
- 展示当前成长概况
- 进入背包
- 进入穿戴
- 进入主线

### 3.3 主线页

状态：

- loading
- success
- empty
- error

交互：

- 选择章节
- 选择关卡
- 选择难度
- 进入出战

### 3.4 出战页

状态：

- loading
- preparing
- success
- error

交互：

- 确认出战角色
- 确认本场目标
- 进入 Battle

### 3.5 Battle 页

状态：

- loading
- success
- settling
- error

显示内容：

- 当前战场目标
- 当前战斗节奏
- 当前 battle context
- 攻击 / 受击 / 击杀 / 掉落 / 结束反馈

交互：

- 推进战斗
- 进入 Settle

### 3.6 Settle 页

状态：

- loading
- settling
- success
- error

显示内容：

- drop_results
- reward_results
- inventory_results
- 本轮后续建议

交互：

- 去背包
- 去穿戴
- 去角色
- 回主线

### 3.7 背包页

状态：

- loading
- success
- empty
- error

交互：

- 查看本轮新增
- 查看 stack items
- 查看 equipment instances
- 跳转穿戴
- 回角色或主线

### 3.8 穿戴页

状态：

- loading
- success
- error
- equip submitting
- unequip submitting

交互：

- 选装备穿戴
- 卸下装备
- 主副武器联动变化后界面立即刷新
- 返回角色或主线

---

## 4. 错误与提示规则

### 4.1 错误类型要区分

至少区分：

- 网络错误
- 认证失效
- 业务失败
- 资源不存在

### 4.2 不静默吞错

任何失败都必须有：

- 明确提示
- 或明确回退动作

### 4.3 提示不替代真实状态

提示文案可以友好，但不能篡改真实结果。

### 4.4 运行期优先保持本地承接

如果旧接口或弱联网能力失败：

- 优先保留当前本地运行时上下文
- 提示玩家当前还能做什么
- 不要把所有页面都退回成“重新请求才可继续”

---

## 5. 奖励与成长显示规则

最少应显示：

- 有首通奖励 / 无首通奖励
- 已领取 / 未领取
- grant_status（若返回）
- 本轮掉落、奖励、入包与新装备承接

说明：

- 现有首通奖励状态仍以现有后端结果或已回流结果为准
- 但结果页、背包页、穿戴页、角色页必须把这些结果继续承接成玩家语言

---

## 6. 当前阶段结论

当前客户端不是先做更重的联网系统，而是先把本地主循环状态收完整。
谁先把页面状态、回流状态和失败承接收好，谁就更接近正式单机刷图成长体验。
