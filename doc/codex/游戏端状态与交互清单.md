# 《山海巡厄录》游戏端状态与交互清单

## 1. 文档目的

本文档用于统一 phase-one 游戏端页面状态、交互边界和最小反馈规范。

---

## 2. 所有页面必须具备的基础状态

每个页面至少必须处理：

1. loading
2. success
3. empty
4. error
5. unauthorized

不允许：
- 只有成功态
- 出错直接白屏
- token 失效后无处理

---

## 3. 页面级状态要求

### 3.1 角色创建页
状态：
- 输入中
- 提交中
- 创建成功
- 创建失败

交互：
- 点击创建后禁用重复点击
- 成功后跳转角色详情
- 失败后提示原因

### 3.2 角色详情页
状态：
- loading
- success
- error
- unauthorized

交互：
- 进入背包
- 进入穿戴
- 进入章节列表

### 3.3 背包页
状态：
- loading
- 有数据
- 空背包
- error

交互：
- 查看 stack items
- 查看 equipment instances
- 跳转穿戴操作

### 3.4 穿戴页
状态：
- loading
- success
- error
- equip submitting
- unequip submitting

交互：
- 选装备穿戴
- 卸下装备
- 主副武器联动变化后界面立即刷新

### 3.5 章节列表页
状态：
- loading
- success
- empty
- error

交互：
- 进入某章节/关卡难度

### 3.6 关卡难度页
状态：
- loading
- success
- error

扩展显示状态：
- has_reward
- has_granted
- grant_status

交互：
- 选择难度进入 battle prepare

### 3.7 战斗准备页
状态：
- preparing
- success
- error

显示内容：
- 角色战斗属性摘要
- monster_list
- battle_context_id

交互：
- 进入结算
- 返回上一页

### 3.8 战斗结算结果页
状态：
- settling
- success
- error

显示内容：
- drop_results
- reward_results
- inventory_results
- first_clear_reward_status

交互：
- 返回章节列表
- 返回角色详情

---

## 4. 错误与提示规则

### 4.1 错误类型要区分
至少区分：
- 网络错误
- 认证失效
- 业务失败
- 资源不存在

### 4.2 不静默吞错
任何失败都必须有：
- 明确提示
- 或明确回退动作

### 4.3 提示不替代真实状态
提示文案可以友好，但不能篡改真实后端结果。

---

## 5. 首通奖励状态显示规则

最少应显示：
- 有首通奖励 / 无首通奖励
- 已领取 / 未领取
- grant_status（若返回）

不要客户端自行推断奖励是否已发放，必须以后端状态为准。

---

## 6. 当前阶段结论

phase-one 客户端不是先做炫酷表现，而是先把状态收完整。
谁先把 loading / error / unauthorized / reward state 收完整，谁的联调成本最低。
