本文档覆盖：

## 3. 数据库设计总原则

### 3.1 分层原则
数据库按以下四层设计：

1. 静态配置层
2. 玩家状态层
3. 产出配置层
4. 关系与承接层

### 3.2 模板与实例分离原则
- 装备模板：`items` + `equipments`
- 装备实例：`inventory_equipment_instances`

### 3.3 随机掉落与固定奖励分离原则
- 掉落：`drop_groups` + `drop_group_items` + `drop_group_bindings`
- 奖励：`reward_groups` + `reward_group_items` + `reward_group_bindings` + `user_reward_grants`

### 3.4 静态配置与玩家状态分离原则
- 职业、物品、装备、怪物、章节、关卡、难度、掉落组、奖励组属于静态配置
- 角色、穿戴槽、堆叠背包、装备实例、发奖记录属于玩家状态或承接记录

### 3.5 长期兼容原则
虽然当前先服务第一期，但所有核心表必须兼容完整游戏长期蓝图：

- 角色长期成长围绕 `characters`
- 装备长期成长围绕 `inventory_equipment_instances`
- 随机掉落扩展围绕 `drop_groups` + `drop_group_bindings`
- 固定奖励扩展围绕 `reward_groups` + `reward_group_bindings` + `user_reward_grants`

定位：一次正式发奖中的逐条奖励对象明细表。

主键：
- `id` bigint unsigned auto_increment

外键：
- `reward_grant_id` → `user_reward_grants.reward_grant_id`
- `item_id` → `items.item_id`
- `equipment_instance_id` → `inventory_equipment_instances.equipment_instance_id` nullable

核心字段：
- `reward_grant_id` bigint unsigned
- `item_id` varchar(64)
- `quantity` int
- `equipment_instance_id` bigint unsigned nullable
- `created_at` datetime

说明：
- 用于记录某次正式发奖实际发了哪些对象
- 对堆叠类物品，按 `item_id + quantity` 记录
- 对装备类奖励，可记录对应创建出的 `equipment_instance_id`
- 该表用于审计、补发核对、后台查询与客诉排查

索引建议：
- `idx_reward_grant_items_grant (reward_grant_id)`
- `idx_reward_grant_items_item (item_id)`

## 8. 关键字段归属原则

### 8.1 共性字段归主表
名称、图标、描述、稀有度、启用状态、排序等共性字段，归对象主表。

### 8.2 专属字段归扩展表
装备行为字段、掉落权重字段、奖励数量字段等，归扩展表或明细表。

### 8.3 玩家状态字段归状态表
等级、经验、属性点、持有数量、穿戴实例、锁定状态、实例成长状态等，归玩家状态层。

### 8.4 计算结果不归核心主表
最终基础属性、最终战斗属性、最终面板值，不直接固化在 `classes` / `characters` / `items` / `equipments` 等核心主表。

### 8.5 高频实例字段显式列化
当前：
- `bind_type`
- `enhance_level`
- `durability`
- `max_durability`
- `random_seed`

都属于结构化字段。

### 8.6 扩展字段只承接低频动态内容
- `characters.extra_context`
- `inventory_equipment_instances.extra_attributes`
- `user_reward_grants.grant_payload_snapshot`

这类字段用于低频、易变化、未完全定型内容，不替代正式高频字段。

### 8.7 来源关系优先走绑定表
- 掉落来源 → `drop_group_bindings`
- 奖励来源 → `reward_group_bindings`

不把来源关系直接塞进组主表或对象主表。

## 11. 当前阶段结论

1. 当前数据结构按静态配置层、玩家状态层、产出配置层、关系与承接层分层  
2. 当前核心表清单固定为 20 张  
3. 装备模板与装备实例已拆分  
4. 角色主表已增加长期兼容预留字段  
5. 掉落与奖励来源必须通过绑定表关联具体来源对象  
6. 当前阶段掉落只和怪物挂钩，副本本身不挂随机掉落  
7. 当前阶段奖励只服务副本不同难度首通固定奖励  
8. 奖励发放必须通过 `user_reward_grants` 做幂等与防重  
9. 奖励发放明细建议通过 `user_reward_grant_items` 留痕  
10. 装备实例同时保留高频结构化字段与 `extra_attributes` 扩展字段  
11. 当前阶段统一采用“主表 + 扩展表”结构思路  
12. 当前阶段统一采用“静态定义与玩家状态分离”的结构思路  
13. 后续新增系统必须在不破坏当前主结构前提下扩展
